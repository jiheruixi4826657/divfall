## 2. 世界觀與故事

### 背景設定
**神殞世界**（Divfall World）是一個曾由神明統治的劍與魔法世界，五百年前神明彼此交戰，最終引發「神殞事件」——神力過載摧毀了大半文明，各地陷入廢墟，殘存神明化為「墮神」繼續主宰世界。

玩家扮演「遺物獵人」，蒐集散落各地的神聖遺物，試圖復甦古老封印以徹底消除墮神威脅。

### 五章節地圖

| 章節 | 名稱 | 地點 | 元素主題 | 視覺主調 |
|------|------|------|----------|----------|
| Ch.1 | 鐵鏽廢都 | 崩塌的神聖帝國都城 | 物理 / 火 | 鏽紅+灰棕 |
| Ch.2 | 腐敗森林 | 神力污染的古老森林 | 木 / 土 | 暗綠+汙紫 |
| Ch.3 | 沉沒聖殿 | 半沉於海中的神殿遺址 | 冰 / 雷 | 深藍+青綠 |
| Ch.4 | 虛空裂縫 | 神殞事件撕裂的次元空間 | 暗 / 光 | 虛空黑+裂縫金 |
| Ch.5 | 神座遺址 | 最後一位墮神的王座 | 全屬性 | 刺眼白金 |

### 故事主線（含真結局）

**序章**：三位獵人相遇於廢都廢墟，各懷目的，決定暫時合作尋找傳說中能封印墮神的「神座核心」。

**Ch.1~4**：逐步探索各地，遭遇墮神爪牙，發現神座核心碎片，同時揭露「神殞事件」的真相——神明並非被外力消滅，而是主動選擇自毀以換取人類文明的存續，但有一位墮神拒絕赴死，竊走了神力獨活至今。

**Ch.5 一般結局**：擊敗最終墮神，封印神座，世界恢復平靜。
**Ch.5 真結局觸發條件**：Ch.1~4 各收集一個「神諭碎片」（BOSS 低概率掉落），在 Ch.5 最終 BOSS 前使用。
**真結局**：三位主角得知若完全消滅最終墮神，其神力將引爆釋放再次毀滅世界。他們選擇將神力封存於自己體內，以凡人之軀化作新的「封印體」，永久守護世界。

## 3. 三位主角背景

### VAREK — 斷神騎
- **職業定位**：物理近戰 + 反制盾牌，高防爆發型
- **背景**：前神聖帝國皇家騎士，神殞事件中目睹皇帝被墮神操控屠城，唯一倖存者。身穿刻有封印符文的碎甲，發誓親手終結墮神時代。
- **核心技能**：神聖護盾（格擋反制）、落裂斬（大範圍倒地）
- **元素**：物理 / 光 / 暗

### LYRA — 術式者
- **職業定位**：五行元素法師，元素連鎖爆發型
- **背景**：古老魔法學院最後的術士，學院被腐敗森林的木系污染毀滅，她蒐集神聖遺物以重建潔淨魔法秩序。性格冷靜理智，但對失去一切的憤怒埋藏極深。
- **核心技能**：元素連鎖（觸發屬性連鎖爆炸）、術式轉換（即時切換主屬性）
- **元素**：火 / 冰 / 雷 / 木 / 土

### KAEL — 影刃者
- **職業定位**：暗系刺客，爆發消失型
- **背景**：來自暗影公會的頂級刺客，接受了「消滅所有墮神」的終身委託。面具從不摘下，無人知曉真實身份。實際上是神殞事件中被墮神吞噬靈魂後又逃脫的倖存者，身體中殘留著黑暗神力碎片。
- **核心技能**：影遁（瞬移殘影）、暗殺標記（累積爆發）
- **元素**：暗 / 物理 / 魔法

## 4. 核心循環

### Roguelite 結構
```
主選單/Hub → 選擇職業 → 進入跑圖
    → 通關關卡 → 三選一卡牌 → 繼續
    → 每5關遇BOSS → 掉落裝備
    → 死亡：卡牌/技能重置，金幣保留（確認後）
    → 回到Hub：天賦永久保留，解鎖卡牌池保留
```

### 關卡結構

| 層級 | 數量 | 說明 |
|------|------|------|
| 章節 | 5 | 各有主題與故事 |
| 關卡/章節 | 50 | 共 250 關 |
| BOSS 關卡 | 每 5 關 1 次 | 共 50 BOSS |
| 一般 BOSS | 45 | Ch.1~4 各9隻，Ch.5 9隻 |
| 章節 BOSS | 5 | 每章第50關 |

### 死亡機制
- **重置**：當前局卡牌、技能、HP、局內金幣
- **保留**：天賦點數（永久）、解鎖卡牌擴充包、裝備（已裝備）
- **金幣**：死亡後彈出確認視窗「保留金幣返回大廳（消耗5%手續費）或放棄」

### 敵人數值縮放公式

```
敵人HP = 基礎HP × (1 + 章節係數 × 0.4) × (1 + 關卡內係數 × 0.08)
敵人攻擊 = 基礎攻擊 × (1 + 章節係數 × 0.35)

章節係數：Ch.1=0, Ch.2=1, Ch.3=2, Ch.4=3, Ch.5=4
關卡內係數：第N關=N-1
```

### BOSS HP 目標

| 章節 | 一般BOSS HP | 章節BOSS HP |
|------|------------|------------|
| Ch.1 | 8,000~12,000 | 45,000 |

| Ch.4 | 50,000~70,000 | 260,000 |
| Ch.5 | 90,000~130,000 | 500,000（真結局：750,000） |

### 傷害計算公式

```
最終傷害 = (攻擊力 × 技能倍率 - 敵人防禦 × 0.5) × 屬性倍率 × 暴擊倍率
最低傷害保底 = 攻擊力 × 技能倍率 × 0.1（穿透防禦保底）
```

## 15. 多人連線系統

### 架構概覽

- 最大2~3人/房間
- 跨平台（Mobile + PC）

## 21. 效能目標

### 幀率目標

| 裝置等級 | 目標幀率 | 最低可接受 |
|----------|---------|-----------|
| 中高階手機（2022+） | 60fps | 55fps |
| 低階手機 | 30fps 省電模式 | 27fps |
| PC | 60fps | 60fps |

### 優化策略

| 優化項目 | 做法 |
|----------|------|
| 特效上限 | 手機最多同時30個粒子效果 |
| 場景載入 | 異步預載下一關卡資源 |
| 音訊 | 懶加載BGM（首次播放才fetch） |
| 網路 | 玩家位置100ms間隔，非即時廣播（降低流量） |
| 渲染 | 螢幕外物件停止更新（Culling） |
| 動畫 | 死亡敵人在動畫結束後即刪除物件 |

### 流暢體驗設計原則
1. **卡頓防護**：卡牌選擇使用CSS動畫，非Canvas渲染
2. **預加載**：章節開始時預加載該章所有資源
3. **輸入響應**：玩家輸入即時更新本地位置（不等待伺服器確認），收到伺服器修正後平滑插值
4. **Hit Stop 精準**：使用 `requestAnimationFrame` 暫停實現，不依賴 `setTimeout` 誤差
5. **省電模式**：電量<20%自動建議切換30fps模式
